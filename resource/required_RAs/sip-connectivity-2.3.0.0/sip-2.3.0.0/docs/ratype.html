RA Type Overview

Introduction

The OpenCloud SIP RA supports the JAIN SIP 1.2 RA Type API, as described in Appendix D of the JAIN SLEE 1.1 Specification. This RA Type is based on the JAIN SIP 1.2 API, with some additional features for SLEE applications.

The SIP RA also supports an additional RA Type, called OCSIP. This extends the JAIN SIP 1.2 RA Type with some useful convenience methods, and header interfaces for 3GPP ISC support.

The RA Type APIs are described in the javadocs included with this distribution.

  • OCSIP 2.2 RA Type API

Migrating from previous versions of the OCSIP RA Type

Applications written for earlier OCSIP RA Types (1.4 and earlier) will need some minor changes to use the new JAIN SIP or OCSIP RA Type. These changes are described here.

SBB Resource Adaptor Interface

In the previous RA Type, the SBB RA Interface was OCSipResourceAdaptorSbbInterface. This interface has been replaced by SleeSipProvider in the standard JAIN SIP 1.2 RA Type. This extends javax.sip.SipProvider, and provides methods for accessing the JAIN SIP factory objects.

To access OpenCloud-specific features and headers, the OCSIP RA Type can be used, and this provides the OCSleeSipProvider interface, which extends SleeSipProvider.

Applications written using the previous RA type must replace all references to OCSipResourceAdaptorSbbInterface with SleeSipProvider or OCSleeSipProvider.

Event Type Identifiers

All event type IDs have been changed in the new RA Type. All events in the JAIN SIP RA Type have the vendor "net.java.slee" (denoting the JAIN SLEE community) and version "1.2". Consult the table in the JAIN SIP 1.2 RA Type Overview Page to see the complete list of events.

Users of OpenCloud-specific events should also check the event table in the OCSIP RA Type Overview Page.
